The project and it’s components

In this file I will give you a deep insight into the code and on how you can can use the different parts of it. As the code is divided into several files (in the 'Extensions' group),
I will explain you file by file, so that if you need only a specific part of the code, you should easily find it.



Here are the main items of my code. If you only need one or a few of them, feel free to scroll down to the part you are interested in, to get everything you need for this item.


Here are the different sections of the code 

  • Declaring and setting up all the variables : GameScene.swift
  • Algorithm to get a random number : Random.swift
  • Set up a timer : Time.swift
  • Create the random boxes from the game and make them move down : CreateBoxes.swift
  • Create borders on the edge of the screen so that the bullet can not fly away : CreateBorders.swift
  • Shoot bullets to a user-defined position : Shoot.swift
  • Handle the touches of the user : Touches.swift
  • Check which nodes collided : Contacts.swift
  • Setting up and ending a game : EndCreateGame.swift
  • Setting up the in game-menu : InGameMenu.swift
  • Setting up the game over-view : GameOverView.swift
  • Setting up the main menu with all its features (jumping ball, several buttons...) : Menu.swift
  • Get the code for the animated pause-button : PlaybackButton.swift
  • Saving and loading data from core data : CoreData.swift
  • Sharing on Facebook and Twitter : Share.swift
  • Display a banner and video ad : Ad.swift

GameScene.swift

This is the main file of the whole project. It declares the class GameScene in which all the functionalities of the game are coded.

In fact, I could have put the whole code of this game into this file, but to guarantee a good overview of the project, I decided to split 

up the code into several files, sorted by categories.

If you would want to add new features to the game, GameScene would be the class in which you would have to write your code, no matter if 

you put it into the file GameScene.swift, or in a separate file like I did. If you use a separate file, write your code like this



import SpriteKit


extension GameScene{

  // write your code here

}



What have i coded in the GameScene.swift file itself?

In this file, I declared all the variables, I would need through the whole game. Besides giving me a good overview of my variables, declaring

them at the beginning, outside any functions, allows me to use them in any function throughout my code.


Furthermore, this file contains the didMoveTo(view:) function.

This function gets called only once : when the app has been opened. It is the place to initialise any variable and

to call the necessary functions to start the game.



Random.swift

There is nothing much to say about this file. It only contains a very simple function which you can call to get a random number of a range



Time.swift

This file contains to two functions to to start a timer a function to update a label (make a countdown). You can easily change the time

interval and the functions called by the timer called every x seconds.



CreateBoxes.swift

This is the file which creates the boxes randomly. Within a for loop from 1-8, 8 boxes are created (or not). To decide whether a box is

created or not, I used the randomNumber function. If in a range from 0-1, it gives me 0, no box is created and we move forward to 

the next position, if it gives me 1,a box is created.

Furthermore, this file contains the algorithm to move the boxes down (the second for loop), and it created the text labels for the boxes to

show how many hit points they have left.



CreateBorders.swift

This again is a more simple file. It creates 4 borders forming a rectangle, so that when the user shoots the bullets, they can not go

off screen.



Shoot.swift

This file contains 3 functions. The first one simply creates a bullet, not doing anything but showing the user from where he is going to shoot

next. The second one gets called by one of the timers to shoot bullets in a regular time interval once to user decided to shoot. It creates a bullet

and uses the last function to get the speed and the direction in which the bullet is going to fly. (receives an impulse in the form of a vector)



Touches.swift

This is the file which handles all the touches given by the user. It handles what to do when the user began touching the screen, when he moved

his finger around and the screen and finally when e ended his touch. This file does not do any extraordinary things, e.g. pressing a button when the

user touched a button. (well... that's still quite impressive isn't it?)



Contacts.swift

The file to handle all crashes. Well, that may sound more spectacular than it actually is. In fact, the function it contains gets called every 

time two nodes collided. What I did then, is checking which nodes collided and reacting accordingly. E.g a bullet collides with a box : The hit points

of the box is reduced by one. (This makes the game look more boring than it actually is...)



EndCreateGame.swift

This file again contains three main functions. The first one resets all the necessary variables(e.g, preloading the video ad, setting the score to 0)

to start a game and actually starts it. The second one checks if the round is over or not(whether there are still bullets flying or not). The third one

 gets called after every round of the game to check whether the user lost or not.



InGameMenu.swift

This file creates the in game-menu. (Yep, this explanation was not necessary). It really does not do anything outrageous, only layout things. The only 

interesting thing might be the function which decided what to when the user presses the pause button.(Pausing the game wouldn't be a badd idea)



GameOverView.swift

This file creates the view once the user has lost the game. In this view he gets to choose whether he wants to look a video to get another chance or not.

Then,  it also contains a function to delete the game over-view, which gets called when the user wanted to get another chance so that he can keep on playing.



Menu.swift

This file is pretty similar to the InGameMenu.swift and GameOverView.swift. It again does only layout things (buttons etc...). There is only one interesting 

thing : the function which creates this cute bouncing bullet in the menu. I you need it, feel free to grab that function.



PlayBackButton.swift

There is nothing much to say about this file. If want to use the same playback button in another project, just copy&paste this file into your project and create

an instance of the playback button like this:


func createPauseButton() {

        pauseButton = PlaybackButton(frame: CGRect(x: 0, y: 5, width: self.frame.width / 10, height: self.frame.width / 10 ))

        pauseButton.contentEdgeInsets = UIEdgeInsets(top: 16, left: 16, bottom: 16, right: 16)

        pauseButton.adjustMargin = 1

        pauseButton.backgroundColor = UIColor.clear

        pauseButton.setButtonColor(UIColor.white)

        pauseButton.addTarget(self, action: #selector(didTapPauseButton(_:)), for: UIControlEvents.touchUpInside)

        self.pauseButton.setButtonState(.playing, animated: false)

        self.scene?.view?.addSubview(self.pauseButton)

    }



GameOverView.swift

This file creates the view once the user has lost the game. In this view he gets to choose whether he wants to look a video to get another chance or not.

Then,  it also contains a function to delete the game over-view, which gets called when the user wanted to get another chance so that he can keep on playing.



CoreData.swift

A simple but maybe interesting file. If you want to save small values, like the high score etc..., you can just copy&paste the two functions this file contains

and replace the variables with the ones you want to be saved in core data. By saving values into core data, they will not get deleted even if the user closes the game

completely.



Share.swift

This file only contains one but very important function. This function contains everything you need to share something on Facebook and Twitter. Simply call this

function whenever you the user to be able to share something directly from your app, et voilà, everything will work just fine.



Ad.swift

This file contains everything you need to show and hide banner ads, rewarded video ads and interstitial ads. To display ads of your own account, read my small guide in the READM.ME file.

Once you did that and replaced your own app-id in the placeholders, this functions will work just fine. Again, if you need them somewhere else, feel free to just copy&paste them!:)
